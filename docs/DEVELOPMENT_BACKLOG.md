# Development Backlog (Roadmap)

This backlog organizes the findings from the `FINAL_PROJECT_REVIEW.md` into actionable, prioritized sprints. It serves as the baseline roadmap for enterprise feature expansion.

---

## Sprint 1: Critical (Security, Architecture, & Workflows)

### 1.1 (COMPLETED) Enforce RBAC & Multi-Tenancy Security
*   **Description**: Consistently apply `RequireRole` decorators and role validation logic across all recently added domains (Finance, Compliance, Integrations).
*   **Priority**: Critical
*   **Estimated Complexity**: Medium
*   **Dependencies**: None
*   **Files/Modules Affected**: `app/domain/*/router.py`, `app/core/deps.py`

### 1.2 (COMPLETED) Implement Cross-Tenant Data Isolation Logic
*   **Description**: Refactor data access filters to securely allow data sharing across tenants (e.g., a Carrier viewing a Settlement generated by a Broker) without exposing broader tenant data.
*   **Priority**: Critical
*   **Estimated Complexity**: High
*   **Dependencies**: Task 1.1 (RBAC)
*   **Files/Modules Affected**: `app/domain/finance/service.py`, `app/domain/compliance/service.py`

### 1.3 (COMPLETED) Centralize Workflow State Machine
*   **Description**: Move imperative state transition logic (e.g., Load from `AVAILABLE` to `COVERED`) into a centralized state machine or workflow engine to ensure robust transition validation.
*   **Priority**: Critical
*   **Estimated Complexity**: High
*   **Dependencies**: None
*   **Files/Modules Affected**: `app/domain/freight/loads/service.py`, `app/domain/freight/shipments/service.py`

### 1.4 (COMPLETED) Enforce Alembic Database Migrations
*   **Description**: Remove `Base.metadata.create_all()` from `main.py` and enforce `alembic upgrade head` for production-safe database migrations.
*   **Priority**: Critical
*   **Estimated Complexity**: Low
*   **Dependencies**: None
*   **Files/Modules Affected**: `backend/app/main.py`, `alembic/`
